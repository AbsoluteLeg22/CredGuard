1️⃣ Common Weak Passwords

Attackers often begin with lists of passwords that are already known to be weak. These are built from previously exposed credentials and still work surprisingly often against reused or predictable passwords.

2️⃣ Dictionary Attacks

Dictionary attacks test real words and predictable variations. A password can appear more complex than it really is if it is still based on a common word with simple numbers or symbols added.

  • Welcome1
  • Summer2026
  • Football!7
  • London123

CredGuard checks for dictionary-style patterns and lowers the rating when a password remains predictable.


3️⃣ Rainbow Tables

A rainbow table is a precomputed collection of password hashes. Instead of testing every possible password, an attacker can compare a leaked hash against known hash values to recover common passwords faster.

Modern systems reduce this risk using salting, but common passwords are still a major weakness when hashes are exposed.

4️⃣ Brute Force vs Intelligent Guessing

Pure brute force tries every combination, but real attackers usually start with smarter methods such as common lists, dictionary words, reused passwords and predictable patterns.


5️⃣ GPU Acceleration & Hash Cracking

GPUs can test extremely large numbers of guesses per second against some hash types. Stronger password hashing methods help slow this down, but weak passwords can still be exposed very quickly.


6️⃣ Real-World Breach Impact

Real breaches give attackers huge datasets of reused and weak credentials. CredGuard demonstrates why both entropy scoring and real-world pattern detection are important when assessing password strength.
